# Development Workflow for Sharpshooter

## Current Workflow: Direct to Main

All commits are pushed directly to the `main` branch. No feature branches or pull requests are required for routine development.

### Process for Changes

1. Make changes to the codebase
2. Run tests: `npm test -- --run`
3. Run e2e tests: `npm run test:e2e`
4. Run lint: `npm run lint`
5. Build: `npm run build`
6. Fix until all green
7. Commit: `git commit -m "feat: description"`
8. Push: `git push origin main`

### Example
```bash
npm test -- --run && \
npm run test:e2e && \
npm run lint && \
npm run build && \
git add -A && \
git commit -m "feat: new feature" && \
git push origin main
```
